# 메서드 오버라이딩

## 정의
- 부모 클래스의 메서드를 자식 클래스에서 재정의함
- 메서드 시그니처(이름, 매개변수)가 동일해야 함
- `@Override` 어노테이션 사용 권장함

## 규칙
- 메서드명, 매개변수 타입/개수/순서 동일해야 함
- 반환타입 동일 (또는 하위 타입 가능)
- 접근제어자 더 좁게 변경 불가함
- 부모보다 더 큰 예외 던질 수 없음

## 예시
```java
class Animal {
    void sound() {
        System.out.println("동물 소리");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("멍멍");
    }
}
```

## 오버로딩 vs 오버라이딩
| 구분 | 오버로딩 | 오버라이딩 |
|------|---------|-----------|
| 위치 | 같은 클래스 | 상속 관계 |
| 매개변수 | 다름 | 동일 |
| 목적 | 편의성 | 재정의 |

## super 키워드
- 부모 클래스 참조에 사용함
- `super.메서드명()`: 부모 메서드 호출
- 오버라이딩된 메서드에서 부모 기능 활용 가능함
